# Unifi Firewall Policy (Resource)

Manages firewall policies using the integration/v1 API (UniFi Network API 9.x+).

> **Important**: This resource requires UniFi Network API 9.x+ and uses the `integration/v1` API endpoints. The legacy `unifi_firewall_rule` resource does not persist on UniFi OS 9.x+.

## Prerequisites

### API Key

1. Navigate to: `https://<your-controller-ip>/network/default/integrations`
2. Click **"Create New Key"** or **"Add API Key"**
3. Give it a name (e.g., "Terraform")
4. Select **"Full Access"** or **"System Administrator"** permissions (required for firewall policies)
5. Click **"Create"** and copy the API key immediately (you won't see it again)
6. Store it securely in your Terraform variables or environment

### Find Your Site UUID

```bash
curl -k -s "https://<controller-ip>/proxy/network/integration/v1/sites" \
  -H "X-API-KEY: <your-api-key>" | jq '.data[] | {name, id}'
```

### Find Your Zone IDs

```bash
curl -k -s "https://<controller-ip>/proxy/network/integration/v1/sites/<site-uuid>/firewall/zones" \
  -H "X-API-KEY: <your-api-key>" | jq '.data[] | {name, id}'
```

## Example Usage

### Basic Zone-to-Zone Policy

```terraform
resource "unifi_firewall_policy" "allow-internal" {
  name                = "Allow Internal Traffic"
  enabled             = true
  action              = "ALLOW"
  allow_return_traffic = true
  source_zone_id      = "<internal-zone-uuid>"  # Replace with your Internal zone UUID
  destination_zone_id = "<internal-zone-uuid>"  # Replace with your Internal zone UUID
  ip_version          = "IPV4"
  logging_enabled     = false
}
```

### IP-to-IP Policy

```terraform
resource "unifi_firewall_policy" "allow-specific" {
  name                = "Allow Specific IPs"
  enabled             = true
  action              = "ALLOW"
  allow_return_traffic = true
  source_zone_id      = "<internal-zone-uuid>"
  source_ips          = ["192.168.1.100", "192.168.1.101"]
  destination_zone_id = "<internal-zone-uuid>"
  destination_ips     = ["192.168.1.200"]
  ip_version          = "IPV4"
  logging_enabled     = true
}
```

### Block Policy

```terraform
resource "unifi_firewall_policy" "block-external" {
  name                = "Block External Access"
  enabled             = true
  action              = "BLOCK"
  allow_return_traffic = false
  source_zone_id      = "<external-zone-uuid>"  # Replace with your External zone UUID
  destination_zone_id = "<internal-zone-uuid>"  # Replace with your Internal zone UUID
  ip_version          = "IPV4"
  logging_enabled     = true
}
```

## Schema

### Required

- `action` (String) The action to take: `ALLOW`, `BLOCK`, or `REJECT`.
- `destination_zone_id` (String) The destination zone UUID.
- `name` (String) The name of the firewall policy.
- `source_zone_id` (String) The source zone UUID.

### Optional

- `allow_return_traffic` (Boolean) Whether to allow return traffic (only for ALLOW action). Defaults to `true`.
- `destination_ips` (Set of String) List of destination IP addresses to match.
- `description` (String) Optional description of the firewall policy.
- `enabled` (Boolean) Whether the firewall policy is enabled. Defaults to `true`.
- `ip_version` (String) IP version: `IPV4`, `IPV6`, or `BOTH`. Defaults to `IPV4`.
- `logging_enabled` (Boolean) Whether to log matching traffic. Defaults to `false`.
- `site` (String) The site UUID to associate the firewall policy with.
- `source_ips` (Set of String) List of source IP addresses to match.

### Read-Only

- `id` (String) The ID of the firewall policy.

## Import

Import is supported using the following syntax:

```shell
# import using site_id and policy_id (comma-separated)
terraform import unifi_firewall_policy.my_policy <site-uuid>,<policy-uuid>
```

Example:
```shell
terraform import unifi_firewall_policy.allow_internal <your-site-uuid>,<your-policy-uuid>
```

## Notes

- When `allow_return_traffic = true` with `action = "ALLOW"`, a return policy is automatically created with `connectionStateFilter = ["ESTABLISHED", "RELATED"]`.
- Zone UUIDs can be retrieved via the API endpoint: `/integration/v1/sites/{siteId}/firewall/zones`
- This resource requires API keys with **Full Access** or **System Administrator** permissions.
- The `source_ips` and `destination_ips` attributes are optional; if omitted, all IPs in the zone match.
